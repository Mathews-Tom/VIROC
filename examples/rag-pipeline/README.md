# RAG pipeline showcase

Linear `pipeline` grammar. Floor-only primitives. Full native parity on all top three.

This is the onboarding example: the boring, canonical left-to-right flow. It walks the **guided VIROC flow** end to end — a user starts from a concept (here a topic brief), the guided planner derives a script and a scene plan, that becomes editable VidIR, `viroc check` validates it, a review surface is inspected, and one Concrete IR compiles deterministically across Manim, HTML, and Remotion before the reproducibility proof closes it out. It uses the `pipeline` grammar, which stays the boring linear-flow grammar; richer composition lives in the `showcase` grammar (see `examples/showcase-composition`), never as `pipeline` special-casing.

## Guided flow

Every committed artifact below is regenerated by the documented commands; the flow is demonstrated, not narrated:

```bash
viroc ingest examples/rag-pipeline/authoring-request.yaml   # concept -> authoring-brief.yaml
viroc plan     examples/rag-pipeline                         # brief -> script.md, scene-plan.yaml, storyboard.vidir.yaml
viroc critique examples/rag-pipeline                         # storyboard -> expected/review/ (script, scene cards, manifest)
viroc compile  examples/rag-pipeline --backend manim         # storyboard -> deterministic backend source
viroc render   examples/rag-pipeline --backend manim         # env-gated render + manifest proof
```

`authoring-request.yaml` is the only hand-authored input (the concept). `authoring-brief.yaml`, `script.md`, `scene-plan.yaml`, and `storyboard.vidir.yaml` are the deterministic `ingest`/`plan` projection of it; `storyboard.vidir.yaml` is the editable VidIR you own and refine. Re-running `ingest`/`plan` is idempotent and byte-stable.

## Scene arc (guided flow)

| Scene | Claim |
|---|---|
| `problem_setup` | A bare model answer still needs the document corpus. |
| `indexing_path` | Documents split into chunks, become embeddings, and land in the vector store. |
| `retrieval_path` | At query time the question probes the vector store for the nearest context. |
| `answer_synthesis` | Retrieved context carries the question into grounded synthesis. |
| `payoff` | Bare prompting falls short; retrieved context produces a grounded answer. |

## Top-three parity

This example uses only the common visual floor (`rect`/`text`/`arrow`), so every top-three backend renders it natively with **zero degradation** — `viroc compile … --backend {manim,html,remotion}` all exit 0 with no `VIR5033` notes. There are no above-floor `code`/`formula` primitives here; the deterministic Manim degradation path is exercised by `examples/showcase-composition` and `examples/viroc-codebase`.
