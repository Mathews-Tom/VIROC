# VIROC codebase showcase

Topic to verified video. Typed IR. Portable renderers.

This is the flagship example: it walks the **guided VIROC flow** end to end and demonstrates it with committed, regenerable artifacts. A user starts from a concept (repo context, a document set, a topic brief), the guided planner derives a script and scene plan, that becomes editable VidIR, `viroc check` repairs it, a review surface is inspected, and one Concrete IR compiles deterministically across Manim, HTML, and Remotion before the reproducibility proof closes it out. It uses the richer `showcase` grammar (non-row grid, fan-out, and comparison compositions), not the single-row `pipeline` ceiling. The acceptance surface is deterministic compile output across the top three backends; render proof stays env-gated and diagnostic-backed.

## Guided flow

Every committed artifact below is regenerated by the documented commands; the flow is demonstrated, not narrated:

```bash
viroc ingest examples/viroc-codebase/authoring-request.yaml   # concept -> authoring-brief.yaml
viroc plan     examples/viroc-codebase                         # brief -> script.md, scene-plan.yaml, storyboard.vidir.yaml
viroc critique examples/viroc-codebase                         # storyboard -> expected/review/ (script, scene cards, manifest)
viroc compile  examples/viroc-codebase --backend manim         # storyboard -> deterministic backend source
viroc render   examples/viroc-codebase --backend manim         # env-gated render + manifest proof
```

`authoring-request.yaml` is the only hand-authored input (the concept). `authoring-brief.yaml`, `script.md`, `scene-plan.yaml`, and `storyboard.vidir.yaml` are the deterministic `ingest`/`plan` projection of it; `storyboard.vidir.yaml` is the editable VidIR you own. Re-running `ingest`/`plan` is idempotent and byte-stable.

## Top-three parity

Every top-three backend renders the common visual floor (`rect`/`text`/`arrow`) natively. The richer `showcase` grammar also emits above-floor `code` and `formula` primitives: HTML and Remotion render them with full fidelity, while Manim renders each as its floor primitive (`rect`) â€” a deterministic degradation that keeps every object's placement and title and is surfaced as a non-blocking `VIR5033` note, never silently omitted. `viroc compile â€¦ --backend manim` therefore exits 0 and prints the degradation notes; `--backend html` and `--backend remotion` compile with no notes.
